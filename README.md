# portfolio_CS-320

This portfolio is an example of software testing using JUnit.
Through this testing, the written functionality is checked to verify that it adheres to the client requirements.
This test helps to ensure quality code which functions as expected, and doesn't allow actions which would violate the requirements.
The client requirements are set during the planning phase of development, and when testing, I break down those requirements into logical components: no null input, phone number must be 10 digits long, and so on.
I then write tests to validate that input data meets the requirements. I write tests with acceptible data to test system functionality. Then I write tests in an attempt to break the requirements, this verifies that incompatible data will be rejected by the system.
It is my intent to write quality, functionally stable software. To do this, I have to plan out the details of the form of the system which will best meet client requirements. This detailed view of the system defines a testing strategy which will keep focus on the quality and functionality of the code.
